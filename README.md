## Preamble

This license exists because the law hasn't caught up with reality.

Traditional open-source licenses (MIT, BSD, GPL, Apache) are built on a single
foundation: **copyright**. The licensor owns a copyright in the work and grants
you permission to use it under certain terms. Remove the copyright and the
entire mechanism collapses — there is nothing to license, nothing to grant,
nothing to restrict.

As of 2025, the United States Copyright Office has consistently held that works
generated entirely by artificial intelligence, without sufficient human
authorship, are **not copyrightable**. Similar positions are emerging in other
jurisdictions, while some (notably the UK) extend statutory protection to
certain "computer-generated works." This means a growing volume of functional,
deployable software exists in a legal patchwork: ownable in one country,
unownable in another, and carrying no formal warranty or liability framework
in either.

**Slop0BSD** is designed for this reality. It is a hybrid instrument — part
waiver, part license, part contract — for code, specifications, documentation,
and other works that are substantially or entirely generated by AI systems.

It operates in three ordered layers:

1. **Waiver.** To the maximum extent permitted by law, the Distributor waives
   all copyright and related rights in the Work.

2. **Fallback License.** To the extent any rights survive that cannot be
   waived, the Distributor grants an unconditional, irrevocable, perpetual,
   worldwide, royalty-free license to use, copy, modify, and distribute the
   Work for any purpose.

3. **Contractual Terms.** Regardless of copyright status, the Distributor and
   Recipient enter into a contract governing warranties, liability, and
   provenance — enforceable under contract law, not copyright.

The name is intentional. "Slop" is the colloquial term for AI-generated
content. We don't run from it. We label it honestly so downstream consumers
can make informed decisions.

## Definitions

**"AI System"** means any machine learning model, large language model, code
generation tool, or automated system that produces output without direct
character-by-character human authorship. This includes but is not limited to
systems such as ChatGPT, Claude, Copilot, Gemini, Codex, open-weight models,
and local inference engines.

**"AI-Generated"** means the Work was produced primarily or entirely by an AI
System, where human involvement was limited to providing prompts, selecting
among outputs, or performing minor edits that do not constitute sufficient
original authorship under applicable copyright law.

**"AI-Assisted"** means a human author made substantial creative, architectural,
or expressive decisions, using an AI System as a tool — analogous to using a
compiler, IDE autocomplete, or reference manual. The human contribution is the
dominant creative element.

**"Distributor"** means the person or entity that makes the Work available
under this license. The Distributor is always a legal person or entity —
never "None."

**"Recipient"** means any person or entity that receives, uses, copies,
modifies, or distributes the Work.

**"Provenance Declaration"** means the structured metadata file describing the
authorship composition of the Work. The canonical format is a YAML file named
`SLOP0BSD.provenance` at the repository root, conforming to the schema defined
in the Slop0BSD Specification. An equivalent machine-readable format (JSON,
TOML, or embedded in package metadata) satisfies this definition provided it
contains the same fields.

**"The Work"** means any source code, object code, specifications, documentation,
configuration, or other material distributed under this license.

## Provenance Declaration

Every project distributed under Slop0BSD **SHOULD** include a Provenance
Declaration. The canonical filename is `SLOP0BSD.provenance` at the repository
root.

### Schema

```yaml
# SLOP0BSD Provenance Declaration
license: Slop0BSD-0.2.0

provenance:
  # Required. One of: ai-generated | ai-assisted | mixed | human-authored
  authorship: ai-generated

  # Required. The entity distributing the Work.
  distributor: "Jane Doe <jane@example.com>"

  # Required. The AI system(s) used.
  generators:
    - name: "Claude"
      version: "Opus 4"
      provider: "Anthropic"

  # Required. Description of human involvement.
  human_role: "Prompting and output selection only"

  # Optional. Percentage estimate of human-authored content (0-100).
  human_content_pct: 5

  # Optional. Origin of the specification or design.
  spec_origin: "ai-generated"

  # Optional. Date of generation (ISO 8601).
  generated: "2026-03-29"

  # Optional. Link to prompt transcript or session log.
  transcript: "https://example.com/session/abc123"

  # Optional. TrustVer-compatible version string.
  trustver: "1.0.0+ai"
```

### Authorship Categories

| Category | Typical Copyright Status | Description |
|---|---|---|
| `ai-generated` | Likely NOT copyrightable (US) | AI produced the work; human involvement limited to prompts and selection |
| `ai-assisted` | Likely copyrightable | Human made dominant creative decisions; AI used as a tool |
| `mixed` | Partially copyrightable | Identifiable human-authored and AI-generated components |
| `human-authored` | Copyrightable | Traditional human authorship; consider a standard license instead |

## Compatibility & Interaction with Other Licenses

### Downstream Compatibility

Because Slop0BSD first waives copyright (Layer 1) and then grants unconditional
permission on any remainder (Layer 2), the Work can be incorporated into
projects under any downstream license — MIT, Apache-2.0, GPL, AGPL,
proprietary, or anything else. There are no copyleft-triggering conditions,
no attribution mandates, and no patent retaliation clauses.

### Upstream Contamination

If a human wrote code under a separate license (e.g., MIT) and an AI system
subsequently modifies it, the human's original contribution retains its
original copyright and license. The AI-generated modifications may be
non-copyrightable, but the composite work carries the most restrictive
applicable license from its copyrightable components. Slop0BSD does not
override pre-existing copyright.

### Copyleft Interaction

Non-copyrightable material cannot be "licensed" under copyleft terms because
there is no copyright to enforce the copyleft condition. Purely AI-generated
Slop0BSD code incorporated into a GPL project does not itself become
GPL-encumbered. The copyleft applies only to the copyrightable portions. This
is analogous to how the GPL does not apply to mathematical formulas, standard
interfaces, or other non-copyrightable elements within a codebase.

**Note:** This interpretation has not been tested in court. Consult legal
counsel for high-stakes applications.

## Rationale & Legal Basis

### The Problem

Traditional open-source licenses assume a single legal reality: the work is
copyrighted, and the author grants you permission to use it. But AI-generated
works create (at least) three possible realities:

1. The work is copyrightable (because a human contributed enough).
2. The work is not copyrightable (because it's purely AI-generated).
3. The copyright status is unclear or jurisdiction-dependent.

A license that only handles scenario (1) is incomplete. A public-domain
dedication that only handles scenario (2) provides no contractual protections.
Slop0BSD handles all three.

### Prior Art

**CC0 1.0 Universal** pioneered the waiver-then-fallback structure. Slop0BSD
adapts this pattern for software and adds the contractual provenance layer
that CC0 lacks. CC0 was designed for a world where copyright exists and
the author wants to disclaim it. Slop0BSD is designed for a world where
copyright may never have existed in the first place.

**0BSD** is a maximally permissive license but asserts copyright in its
header ("Copyright (c) ..."). For purely AI-generated works, this assertion
may be legally incoherent. Slop0BSD replaces the copyright assertion with a
distributor declaration.

### Key Authorities (United States)

- **Thaler v. Perlmutter (D.D.C. 2023):** AI-generated works without human
  authorship are not copyrightable.
- **U.S. Copyright Office, Registration Guidance (Feb 2023, updated 2025):**
  Works generated by AI require human authorship for registration.
- **Kashtanova / Zarya of the Dawn (2023):** Individual AI-generated images
  denied copyright; human-selected arrangement granted thin copyright.
- **ProCD v. Zeidenberg (7th Cir. 1996):** Shrinkwrap contracts enforceable
  where there is notice and opportunity to reject. Supports enforceability
  of Section 3.

### Key Authorities (International)

- **UK CDPA 1988, §9(3):** Provides copyright for "computer-generated works"
  where the author is "the person by whom the arrangements necessary for the
  creation of the work are made." This may extend to some AI-generated works,
  making Section 2 (Fallback License) operative in UK jurisdictions.
- **EU AI Act (2024):** Requires transparency and labeling of AI-generated
  content. Slop0BSD's Provenance Declaration is compatible with and supports
  these requirements.

## Design Decisions

### Why the Distributor, Not the Author?

Traditional licenses name the copyright author as the grantor. But for
AI-generated works:
- The AI system cannot be a legal person.
- The prompter may not have contributed enough for authorship.
- "Copyright (c) None" is legally meaningless — no person is making the grant.

Slop0BSD names the **Distributor** — the person or entity who packages and
releases the Work — as the operative party. The Distributor is always a real
legal entity who can waive rights (Section 1), grant licenses (Section 2),
and enter contracts (Section 3).

### Why Incentive-Based Provenance, Not Mandatory Retention?

Traditional licenses use copyright to enforce conditions ("you must retain
this notice"). If the work isn't copyrightable, that enforcement mechanism
is absent. Requiring retention without a mechanism to enforce it creates a
false obligation that erodes trust in the license.

Slop0BSD instead uses a **conditional benefit**: the warranty disclaimer and
liability limitation in Section 3 protect only distributions that preserve
provenance. Strip the provenance, and you redistribute "naked" — with full
exposure to warranty and liability claims. This is self-enforcing because:

- It doesn't depend on copyright for enforcement.
- It creates a natural economic incentive to preserve provenance.
- It doesn't create unenforceable mandates or false obligations.
- Downstream redistributors make a rational choice, not a coerced one.

### Why Not Just Use CC0?

CC0 is excellent for what it does, but it was designed for creative works in
a world where copyright exists and the author wants to disclaim it. It lacks:
- AI-specific provenance infrastructure.
- Recognition that the work may *never have been copyrightable*.
- Contractual provisions tailored to software distribution (generator/prompter
  liability, provenance accuracy representations).
- Integration with software ecosystem tooling (TrustVer, SPDX identifiers,
  package metadata).

CC0's fallback license also doesn't address the "three jobs at once" problem
because it doesn't need to — CC0 assumes copyright exists and then destroys it.
Slop0BSD starts from the possibility that copyright never existed.

## FAQ

**Q: Why not just use 0BSD or MIT?**

A: Because they assert copyright ownership ("Copyright (c) ...") and grant
permission *from* that copyright. If the work isn't copyrightable, those
assertions are legally incoherent. More importantly, they provide no
provenance framework, and their warranty disclaimers may be unenforceable
if the underlying copyright assertion fails — the entire instrument depends
on copyright existing.

**Q: Can someone take my Slop0BSD project and sell it?**

A: Yes. Section 1 waives copyright. Section 2 grants unconditional permission
on any remainder. If the work is truly AI-generated and non-copyrightable,
anyone can do anything with it regardless of this license. Slop0BSD doesn't
pretend otherwise.

**Q: What if I strip the provenance and redistribute?**

A: You can. Nobody will sue you for it (there's no enforceable obligation to
retain it). But you lose the warranty disclaimer and liability limitation from
Section 3. You're redistributing "naked" — if the code breaks something,
you have no contractual shield.

**Q: Does the provenance representation create liability for distributors?**

A: Section 3.3(a) is a "best knowledge and belief" representation, not an
absolute warranty. If you accurately describe what you know about the work's
origins, you're protected. If you knowingly falsify provenance (e.g., claiming
code is AI-generated when you copied it from a GPL project), Section 3.3(c)
creates liability for that specific misrepresentation.

**Q: What about jurisdictions where AI-generated works ARE copyrightable?**

A: Section 1 waives that copyright. Section 2 provides a fallback license if
the waiver is incomplete. Section 3 applies regardless. The license degrades
gracefully across legal regimes. In the UK under CDPA §9(3), the Distributor
who "made the arrangements" for the AI generation may hold copyright —
Sections 1 and 2 handle this cleanly.

**Q: Is this license OSI-approved?**

A: No. The OSI's Open Source Definition assumes copyrightable works and
evaluates licenses against that assumption. Slop0BSD explicitly contemplates
non-copyrightable material and includes contractual provisions that go beyond
traditional copyright licensing. We believe the framework needs to evolve.

**Q: Can I use Slop0BSD for a project where I wrote the spec but AI wrote
all the code?**

A: Yes — this is a primary use case. The Provenance Declaration has fields
for both `spec_origin` and `authorship`. Your spec may carry its own copyright;
the generated code likely does not. Slop0BSD handles this cleanly. Consider
licensing the spec separately if it has significant copyrightable content.

**Q: What's the SPDX identifier?**

A: Slop0BSD does not yet have an SPDX-listed identifier. Until it does, use
`LicenseRef-Slop0BSD-0.2.0` in SPDX expressions. We intend to submit for
SPDX listing once the license text stabilizes.

## TrustVer Integration

Projects using both Slop0BSD and the [TrustVer](https://thirdkey.ai/trustver)
versioning specification can embed authorship provenance directly in version
strings:

```
1.0.0+ai        # Fully AI-generated
1.2.0+hrai      # Mixed human and AI authorship
2.0.0+hr        # Human-authored (consider a different license)
```

The `trustver` field in the Provenance Declaration links the two systems.
