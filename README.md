## Preamble

This license exists because the law hasn't caught up with reality.

Traditional open-source licenses (MIT, BSD, GPL, Apache) are built on a single
foundation: **copyright**. The licensor owns a copyright in the work and grants
you permission to use it under certain terms. Remove the copyright and the
entire mechanism collapses — there is nothing to license, nothing to grant,
nothing to restrict.

As of 2025, the United States Copyright Office has consistently held that works
generated entirely by artificial intelligence, without sufficient human
authorship, are **not copyrightable**. Similar positions are emerging in the EU,
UK, and other jurisdictions. This means a growing volume of functional,
deployable software exists in a legal gray zone: it cannot be owned, but it also
carries no warranty, no attribution trail, and no formal liability framework.

**Slop0BSD** is designed for this reality. It is a license for code, specifications,
documentation, and other works that are substantially or entirely generated by AI
systems. It operates on two parallel tracks:

1. **To the extent the work IS copyrightable** (because a human made meaningful
   creative choices in prompting, selecting, arranging, or editing the output),
   this license grants maximally permissive rights — equivalent to the Zero-Clause
   BSD license (0BSD).

2. **To the extent the work is NOT copyrightable** (because it is purely
   AI-generated output with no qualifying human authorship), this license serves
   as a **factual declaration** of that status, a contractual warranty disclaimer,
   and a liability limitation — none of which require copyright to be enforceable.

The name is intentional. "Slop" is the colloquial term for AI-generated content.
We don't run from it. We label it honestly so downstream consumers can make
informed decisions.

## Definitions

**"AI System"** means any machine learning model, large language model, code
generation tool, or automated system that produces output without direct
character-by-character human authorship. This includes but is not limited to
ChatGPT, Claude, Copilot, Gemini, Codex, open-weight models, and local
inference engines.

**"AI-Generated"** means the work was produced primarily or entirely by an AI
System, where human involvement was limited to providing prompts, selecting
among outputs, or performing minor edits that do not constitute sufficient
original authorship under applicable copyright law.

**"AI-Assisted"** means a human author made substantial creative, architectural,
or expressive decisions, using an AI System as a tool — analogous to using a
compiler, IDE autocomplete, or reference manual. The human contribution is the
dominant creative element.

**"Provenance Declaration"** means the `SLOP0BSD.provenance` file or equivalent
metadata (see §Provenance below) that accompanies the work and describes the
authorship composition.

**"The Work"** means any source code, object code, specifications, documentation,
configuration, or other material distributed under this license.

## Provenance Declaration

Every project distributed under Slop0BSD **SHOULD** include a provenance file.
The canonical filename is `SLOP0BSD.provenance` at the repository root.

### Format

```yaml
# SLOP0BSD Provenance Declaration
license: Slop0BSD-0.1.0

provenance:
  # One of: ai-generated | ai-assisted | human-authored | mixed
  authorship: ai-generated

  # The AI system(s) used
  generators:
    - name: "Claude"
      version: "Opus 4"
      provider: "Anthropic"

  # Human involvement description
  human_role: "Prompting and output selection only"

  # Optional: the prompter / curator
  prompter: "Jane Doe <jane@example.com>"

  # Optional: percentage estimate of human-authored content
  human_content_pct: 5

  # Was the specification also AI-generated?
  spec_origin: "ai-generated"

  # Date of generation
  generated: "2026-03-29"

  # Optional: link to prompt transcript or session
  transcript: "https://example.com/session/abc123"

  # Optional: TrustVer-compatible version with authorship tag
  trustver: "1.0.0+ai"
```

### Authorship Categories

| Category | Copyright Status | Description |
|---|---|---|
| `ai-generated` | Likely NOT copyrightable | AI produced the work; human involvement limited to prompts and selection |
| `ai-assisted` | Likely copyrightable | Human made dominant creative decisions; AI used as a tool |
| `mixed` | Partially copyrightable | Identifiable human-authored and AI-generated components |
| `human-authored` | Copyrightable (use a normal license) | Traditional human authorship; Slop0BSD is not the right license |

## Compatibility & Interaction with Other Licenses

### Slop0BSD code can be incorporated into any project

Because Slop0BSD either grants unconditional permission (if copyrightable) or
declares non-copyrightable status (if AI-generated), the work is compatible
with all downstream licenses: MIT, Apache-2.0, GPL, AGPL, proprietary, or
anything else. There is nothing to comply with beyond retaining the license
text and provenance file.

### Other-licensed code cannot become Slop0BSD retroactively

If a human wrote code under MIT and an AI system then modifies it, the human's
original contribution retains its MIT copyright. The AI-generated modifications
may be non-copyrightable, but the composite work carries the most restrictive
applicable license. Slop0BSD does not override pre-existing copyright.

### Interaction with copyleft

Because non-copyrightable material cannot be "licensed" under copyleft terms
(there is no copyright to enforce the copyleft condition), purely AI-generated
Slop0BSD code that is incorporated into a GPL project does not itself become
GPL-encumbered. The copyleft applies only to the copyrightable portions. This
is analogous to how the GPL does not apply to facts, mathematical formulas, or
other non-copyrightable elements within a codebase.

**Note:** This is the authors' interpretation and has not been tested in court.
Consult legal counsel for high-stakes applications.

## Rationale & Legal Basis

### United States

- **Thaler v. Perlmutter (D.D.C. 2023):** Affirmed that AI-generated works
  without human authorship are not copyrightable.
- **U.S. Copyright Office, Copyright Registration Guidance (Feb 2023):**
  Works generated by AI require human authorship for registration; purely
  AI-generated content is not registrable.
- **Kashtanova / Zarya of the Dawn (2023):** Individual AI-generated images
  were denied copyright; human-selected arrangement was granted thin copyright.

### Implications

If a work is not copyrightable, then:
- No one can own it.
- No one can exclusively license it.
- No one can enforce license conditions via copyright.
- **But** contractual terms (warranty disclaimers, liability limitations,
  provenance representations) remain enforceable between parties under
  contract law.

Slop0BSD is designed to work correctly in both worlds: the world where AI
output is copyrightable (granting maximum permission) and the world where it
isn't (providing contractual protections that don't rely on copyright).

## FAQ

**Q: Why not just use 0BSD or Unlicense?**

A: Because they assert copyright ownership ("Copyright (c) ...") and grant
permission *from* that copyright. If the work isn't copyrightable, those
assertions are legally incoherent. Slop0BSD is honest about the ambiguity
and provides the dual-track mechanism — plus the provenance system that
downstream consumers actually need.

**Q: Can someone take my Slop0BSD project and sell it?**

A: Yes. If the work is truly AI-generated and non-copyrightable, *anyone*
can do *anything* with it — they don't even need this license's permission.
If the work has copyrightable human elements, this license grants that
permission anyway. Either way: yes.

**Q: Does the provenance declaration have legal force?**

A: The declaration itself is a factual representation made under the
contractual framework of the license. If a distributor knowingly
misrepresents provenance (e.g., claiming code is AI-generated when it was
copied from a copyrighted project), that could constitute fraud or breach
of contract. The provenance system is designed for transparency, not
enforcement.

**Q: What about jurisdictions where AI-generated works ARE copyrightable?**

A: In those jurisdictions, Slop0BSD operates as a maximally permissive
license — functionally equivalent to 0BSD. The provenance declaration
still serves its transparency purpose. The license is designed to degrade
gracefully across legal regimes.

**Q: Can I use Slop0BSD for a project where I wrote the spec but AI wrote
all the code?**

A: Yes — this is a primary use case. The provenance declaration has fields
for both `spec_origin` and `authorship` so you can accurately describe the
pipeline. Your spec may carry its own (thin) copyright; the generated code
likely does not. Slop0BSD handles this cleanly.

**Q: Is this license OSI-approved?**

A: No. The OSI's Open Source Definition assumes copyrightable works. A
license that explicitly contemplates non-copyrightable material operates
outside the traditional open-source framework. We believe this is a feature,
not a bug — the framework needs to evolve.

## TrustVer Integration

Projects using both Slop0BSD and the [TrustVer](https://thirdkey.ai/trustver)
versioning specification can embed authorship provenance directly in version
strings:

```
1.0.0+ai        # Fully AI-generated
1.2.0+hrai      # Mixed human and AI authorship
2.0.0+hr        # Human-authored (consider a different license)
```

The `trustver` field in the provenance declaration links the two systems.
